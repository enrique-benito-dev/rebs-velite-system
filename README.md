## 🔐 Sistema de permisos por roles

Cada base de datos tiene acceso controlado por subpágina. El equipo solo ve lo que necesita para su función — sin ruido, sin acceso cruzado no autorizado.

| Rol | Acceso |
|-----|--------|
| Admin | Total — todas las BDs + Seguridad completa |
| Notion Manager (EB) | Todas las BDs operativas — edición completa |
| Director de Área | BDs de su área + edición de las propias |
| Empleado Operativo | Tareas, Hitos, Equipo — solo sus registros |
| Colaborador Externo | Solo la BD de su proyecto específico |

---

## 💡 Decisiones de diseño clave

**Estaciones de trabajo personales** — cada miembro del equipo tiene su propio panel de entrada con solo la información relevante para su rol. De 50+ páginas del sistema a 1 vista operativa por persona.

**Diseño no destructivo** — las bases de datos nunca se eliminan, se archivan. El historial del sistema es completo y auditable en cualquier momento.

**Escalabilidad por rol** — añadir un nuevo miembro es asignarle permisos en las BDs correspondientes. El sistema no requiere reconfiguración.

**[SYSTEM] como núcleo técnico separado** — toda la arquitectura, seguridad y administración está aislada del trabajo operativo. El equipo opera sin ver la infraestructura que lo sostiene.
